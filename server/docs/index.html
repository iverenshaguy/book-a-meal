    Introduction

    Welcome to the Book A Meal API documentation! This is the API used by the Book A Meal web interface. This is version 1 of the API.

    BASE URL: https://book-a-meal-sivy.herokuapp.com/api/v1/

    Authentication

    An authorization token is required to access all protected endpoints. Only the Login and Signup endpoints do not require an authorization token. The Login and Signup endpoints are used to generate Authorization Tokens.

    Authorization tokens expire after 48 hours after they are generated.

    BookAMeal expects a token to be included in all protected API requests to the server in a header that looks like the following:

    Authorization: averyveryverylong.jsonwebtoken.requiredforauthentication

    You must replace averylong.jsonwebtoken.requiredforauthentication with your personal Authorization Token.

    • Endpoint: /meals
    • Verb: POST

    title string Required. Unique Meal title

    Mark Order Item as Delivered

    Marks order item(s) belonging to caterer as delivered by the caterer. Once an order item is marked as delivered, the system checks all order items under that order to see if they have been delivered. If yes, the system changes the order status to delivered else, order status is left as pending

    Request Body

    {
  "delivered": true
}

    HTTP Request

    • Endpoint: /orders/:orderId/deliver
    • Verb: POST

    URL Parameters

    Parameter Description
    orderId The ID of the order to be modified

    Request Body Parameters

    Parameter Format Details Description
    delivered boolean Required Notifies system that all order items belonging to that caterer in the order have been delivered
